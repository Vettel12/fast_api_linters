# 📖 API Кулинарной книги (Recipe App)

Асинхронный сервис на **FastAPI** и **SQLAlchemy 2.0** для хранения и просмотра рецептов с автоматическим счетчиком популярности.

## 🚀 Особенности
- **Асинхронность**: Используется `aiosqlite` и `AsyncSession` для высокой производительности.
- **Умная сортировка**: Рецепты в списке автоматически сортируются по количеству просмотров, а затем по времени приготовления.
- **Счетчик просмотров**: Каждое открытие детального рецепта (`GET /recipes/{id}`) автоматически увеличивает его популярность в базе.
- **Полная документация**: Все модели и поля аннотированы для удобства фронтенд-разработки.
- **Строгая типизация**: Проект настроен под `mypy --strict`, что гарантирует надежность кода.

## 🛠 Технологии
- **FastAPI** (v0.109+)
- **SQLAlchemy** (v2.0+) — асинхронный ORM
- **Pydantic v2** — валидация данных
- **SQLite** — база данных (драйвер `aiosqlite`)
- **Pytest** — автоматизированное тестирование
- **Ruff & Mypy** — линтинг и проверка типов

# 🏁 Быстрый запуск

## 1. Клонируйте проект и создайте окружение:
```bash
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
```

## 2. Установите зависимости:
```bash
pip install -r requirements.txt
```

## 3. Запустите сервер:
```bash
uvicorn app.main:app --reload
```

# 🛡️ Контроль качества (Linters & CI)

В проекте настроены современные инструменты проверки кода. Конфигурация хранится в `pyproject.toml`.

## ⚙️ Настройки инструментов

### Ruff (Линтинг и форматирование)
Заменяет Flake8, isort и Black.
- **line-length = 88**: Максимальная длина строки (стандарт Black).
- **select**: Включены проверки ошибок (`E`, `W`), логических ошибок (`F`), сортировки импортов (`I`), поиска багов (`B`) и сложности кода (`C90`).

### Mypy (Проверка типов)
- **strict = true**: Максимально строгий режим. Все аргументы и возвращаемые значения должны быть типизированы.
- **plugins**: Подключены плагины `pydantic.mypy` и `sqlalchemy.ext.mypy.plugin`, чтобы проверщик понимал магию ORM и схем.
- **disallow_untyped_decorators = false**: Исключение для удобной работы с декораторами FastAPI.

### Pytest (Тестирование)
- **asyncio_mode = "auto"**: Позволяет легко писать асинхронные тесты без лишних декораторов.
- **asyncio_default_fixture_loop_scope = "session"**: Оптимизация ресурсов при запуске тестов.

## 🏠 Локальная проверка (Offline)

Перед отправкой кода в репозиторий рекомендуется запустить проверки локально:

1. **Проверка логики и импортов (Ruff)**:
   ```bash
   ruff check .
   ```

2. **Проверка форматирования (Ruff Format)**:
   Чтобы увидеть, что нужно исправить:
   ```bash
   ruff format --check .
   ```
   Чтобы автоматически исправить формат:
   ```bash
   ruff format .
   ```

3. **Проверка типов (Mypy)**:
   ```bash
   mypy .
   ```

4. **Запуск тестов**:
   ```bash
   pytest -v
   ```

## 🤖 Автоматическая проверка (GitHub Actions CI)

Настроен процесс непрерывной интеграции (CI), который автоматически запускается при каждом пуше в ветку `main` или при создании Pull Request.

**Что делает CI:**
1. **Checkout**: Копирует ваш код с GitHub.
2. **Setup Python**: Устанавливает Python 3.13.
3. **Install**: Скачивает все зависимости проекта.
4. **Ruff Check**: Проверяет код на ошибки и неправильные импорты.
5. **Ruff Format**: Проверяет, что код отформатирован красиво.
6. **Mypy**: Гарантирует отсутствие ошибок типизации.
7. **Tests**: Запускает все тесты Pytest.

> Если хоть одна проверка не пройдет, код не будет считаться валидным, и вы увидите красный крестик ❌ во вкладке *Actions* вашего репозитория. Зеленая галочка ✅ означает, что код чист и работает корректно.

# 🔌 API Эндпоинты

* **GET /recipes** — Получить список всех рецептов (отсортированы по популярности).
* **GET /recipes/{id}** — Детальная информация + инкремент просмотров.
* **POST /recipes** — Создать новый рецепт.
